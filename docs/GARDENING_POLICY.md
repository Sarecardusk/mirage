# Docs Gardening Policy

> Last Updated: 2026-04-05

本策略用于维护 `docs/` 的边界文档质量，避免文档漂移。

## 维护原则

- 核心硬约束只在权威文档定义一次。
- 其他文档通过链接引用，不复制全文。
- 文档术语必须与实现保持一致。
- 无行为变化时不做”装饰性改写”。
- 权威文档头部保持 `> Last Updated: YYYY-MM-DD` 标记，修改时同步更新。

## 权威归属

- 架构与职责边界：`DESIGN.md`
- 可靠性与失败路径：`RELIABILITY.md`
- 安全边界：`SECURITY.md`
- Harness 执行语义：`HARNESS.md`
- 兼容性策略：`COMPATIBILITY.md`
- 测试证据策略：`TEST_STRATEGY.md`
- 诊断与可观测性：`OPERATIONS.md`
- 工作约定与协作流程：`AGENTS.md`

## 触发更新条件

- 发生跨边界接口变更。
- 发生数据模型或迁移策略变更。
- 发生失败处理策略变更。
- 发生安全拒绝策略变更。
- 发生公开契约兼容性策略变更。
- 发生测试门禁或诊断字段变更。
- 发生工程工作流或协作约定变更。

## 提交流程建议

- 每完成一个小文档块就进入一次审查点。
- 优先小步提交，确保每次提交可独立审阅。
- 若改动触发多个权威文档，单次提交内必须同步更新对应索引文档。
