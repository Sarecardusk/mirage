# Mirage v1 Product Spec (Boundary Slice)

本规格只记录 v1 期已经锁定的底层实现边界。

## v1 固定约束

- 核心实体为 `Theme Card`。
- 默认作用域模型为 `单用户单工作区 + 同 Theme Card 下多 Session`。
- 同一卡片共享设定；会话上下文与记忆按 Session 隔离。
- 契约真源在 Rust，前端手写 `Zod` 做边界校验。
- 主存储为 `SurrealDB 3.x (SurrealKV)`，二进制资产走 bucket storage。
- `Theme Card` 必带 `schemaVersion` 并支持显式迁移。
- `Profile` 在 v1 仅为保留扩展容器，默认单一且不对用户暴露。

## v1 非目标

- 不建设云端中转 `LLM Gateway` 服务。
- 不引入云账号中心与云托管同步。
- 不承诺 SillyTavern 角色卡格式兼容。

## 验收基线

- 迁移成功可读、失败可追踪。
- 检索与模型失败均可降级展示。
- 安全失败场景全部硬拒绝。
