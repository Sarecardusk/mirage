# PLANS

> Last Updated: 2026-04-05

本文件定义 Mirage 的 ExecPlan 元规范入口。

## 目标

- 统一“方案 -> 审核 -> 分解 -> 执行 -> 复盘”的节奏。
- 保证涉及高重构成本边界的改动有计划、可追踪、可回滚。

## 目录约定

- 进行中计划：`docs/exec-plans/active/`
- 已完成归档：`docs/exec-plans/completed/`
- 技术债清单：`docs/exec-plans/tech-debt-tracker.md`

## 最小流程

1. 在 `active/` 基于模板创建计划。
2. 标注边界影响（设计/可靠性/安全）。
3. 审批通过后执行，并补充验证结果与意外发现。
4. 完成后归档到 `completed/`。

## 审批规则

- **人机协作模式**（默认）：步骤 2 完成后须经人工审批，方可进入步骤 3 执行。
- **AI 独立模式**：对 `Planned` 状态（尚无实现）的 MH/RC/TD 约束，AI 可自审批并执行。涉及 HR 级约束变更的计划，即使约束处于 Planned 状态，也必须标记为需人工审查。
- **时效**：计划创建后应在同一工作会话内完成审批。若需跨会话延续，须在计划中记录延期原因。
